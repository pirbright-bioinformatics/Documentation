# Guidelines for Efficient Storage Management in scRNA-seq Analysis

## 1. Purpose and Scope

Single-cell RNA sequencing (scRNA-seq) analyses generate large volumes of intermediate and output data, often resulting in redundant storage and inefficient use of HPC resources. This guideline outlines best practices for managing, compressing, and removing intermediate files while maintaining reproducibility and long-term usability of results.

---

## 2. Key Principles

### 2.1 Reproducibility over Redundancy

Retain sufficient information to reproduce results (e.g., raw data, reference files, scripts, and final processed objects), rather than all intermediate outputs.

### 2.2 Tiered Storage Strategy

Organise data into tiers based on importance and access frequency:

- **Tier 1 (Essential, Long-Term):**
  - Raw data
  - Metadata
  - Final processed objects

- **Tier 2 (Reproducible Intermediates):**
  - Key count matrices
  - QC summaries

- **Tier 3 (Temporary):**
  - Alignment files
  - Temporary pipeline outputs

### 2.3 Compression and Format Optimisation

Use compressed and efficient formats whenever possible, including:

- `.h5`
- `.h5ad`
- `.rds`
- `.qs`
- `.mtx.gz`

---

## 3. Storage Impact by File Type

### 3.0 Raw Data (Must Retain)

#### BCL Files 

**Action:**

- Output by Sequencer, these files are typically stored within the `RDS-archive` file system on HPC.
- Use symbolic links or read-links to access them where possible.
- Avoid unnecessary copies in `RDS-ephemeral`.

---

### 3.1 Demultiplexed Data (Retain)

#### FASTQ Files (`*.fastq.gz`)

- Generated during demultiplexing (e.g., `cellranger mkfastq`)
- Often among the largest files generated (>100 GB per sequencing run)
- Typically stored within the `RDS-ephemeral` file system on HPC

---

### 3.2 Alignment Files (Major Storage Burden)

#### BAM Files (`*.bam`)

- Frequently exceed 10 GB per sample
- Generated during mapping (e.g., `cellranger count`)

**Recommendations:**

- Remove BAM files after successful verification of mapping and quantification.
- Retain only:
  - Alignment summary statistics
  - Pipeline logs

---

### 3.3 Quantification Outputs (Usually Retain)

**Examples:**

- Gene-barcode matrices:
  - `matrix.mtx.gz`
  - `features.tsv.gz`
  - `barcodes.tsv.gz`
- HDF5 format:
  - `filtered_feature_bc_matrix.h5`

**Recommendations:**

- Prefer `.h5` or `.h5ad` formats for compact storage.

---

### 3.4 Intermediate Pipeline Outputs (Often Disposable Once Project Complete)

**Examples:**

- Archived analysis outputs generated by previous versions of analytical pipelines.
- Multiple `.RData` files generated during:
  - Method development
  - Testing
  - Debugging
  - Reproducibility assessment

**Recommendations:**

- Delete after project completion.
- Automate cleanup using workflow management systems and scripts.
- Remove redundant and obsolete versions.

---

### 3.5 Final Analysis Objects (Retain, but Optimise)

**Examples:**

- R objects: `.rds`
- Python objects: `.h5ad` (Scanpy/AnnData)

**Recommendations:**

- Retain final processed objects only, including:
  - Post-QC data
  - Normalised datasets
  - Annotated datasets

- Use compression:

#### R

```r
saveRDS(..., compress = "xz")
```

or

```r
qs::qsave()
```

#### Python

```python
adata.write(..., compression="gzip")
```

---

### 3.6 Derived Outputs (Selective Retention)

**Examples:**

- Figures (`.png`, `.pdf`)
- Tables (`.csv`, `.tsv`)
- Marker gene results

**Recommendations:**

- Retain final publication-quality figures and summary tables.
- Remove iterative and debugging outputs.

---

## 4. Recommended Retention Matrix

| File Type | Typical Size | Retain? | Notes |
|------------|-------------|----------|--------|
| FASTQ (`.fastq.gz`) | Large |  Yes | Consider deletion after manuscript acceptance and compliance with institutional retention requirements |
| BAM | Very Large |  No | Delete after successful verification |
| Filtered Count Matrix | Moderate |  Yes | Core dataset |
| Raw Count Matrix | Moderate |  No (unless essential) | Retain only if required for QC or re-analysis |
| Temporary Pipeline Files | Variable |  No | Delete after project completion |
| Seurat / AnnData Objects | Moderate |  Yes | Retain final version only |
| Logs and QC Summaries | Small |  Yes | Essential for reproducibility |

---

## 5. Practical Example Workflow

After running a pipeline (e.g., Cell Ranger or STARsolo):

### Step 1: Verify Outputs

Review:

- QC metrics
- Cell counts
- Mapping rates

### Step 2: Retain

- `filtered_feature_bc_matrix.h5`
- `metrics_summary.csv`
- Final Seurat object
- Final AnnData object

### Step 3: Delete

- Temporary directories
- Intermediate versions
- Duplicate matrices
- BAM files

### Step 4: Archive

- Move BCL files to long-term storage (`RDS-archive`)

---

## 6. Automation and Policy Recommendations

### Automated Cleanup

Implement automated cleanup procedures after completion of each scRNA Bioinformatics Support Project (BSP).

### Quota Monitoring

Use storage quota monitoring tools to:

- Record storage consumption
- Track project-level resource usage
- Identify opportunities for storage optimisation

### Documentation

Maintain a `README` or metadata file for each completed project documenting:

- Data locations
- Retained files
- Deleted intermediate files
- Archive locations
- Pipeline versions used

---

## 7. Summary

Efficient scRNA-seq data management in HPC environments requires balancing storage constraints with reproducibility requirements. By prioritising the retention of raw data, final processed objects, and essential metadata—while removing or compressing large intermediate files such as BAMs and temporary pipeline outputs—research groups can substantially reduce storage usage without compromising analytical integrity.

Future improvements may include:

- Standardised data schemas
- Automated lifecycle management
- Integration with institutional data management platforms
- Improved archival and retrieval workflows

These measures will further streamline storage management and improve accessibility for large-scale, multi-sample scRNA-seq projects.
